Скачайте единственный двоичный файл на ваш сервер и начните службу всего за несколько простых шагов.

## Требования

- Любая операционная система на базе Linux. Мы рекомендуем использовать Fedora 42.
- x86 64-битный процессор
- Подключение к Интернету для периодической проверки лицензии (подключается к `ls4.cesbo.com`)

:::note
В этом руководстве используется команда **curl** для загрузки файлов. Большинство дистрибутивов Linux имеют предустановленный **curl**.
:::

## Установка Alta

<Steps>

1. **Откройте консоль вашего сервера и загрузите двоичный файл**

    ```sh
    curl -Lo /usr/bin/alta https://cesbo.com/alta-latest
    ```

2. **Установите разрешение на выполнение**

    ```sh
    chmod +x /usr/bin/alta
    ```

3. **Установка завершена**

    Вы можете проверить установленную версию Alta, выполнив:

    ```sh
    alta version
    ```

</Steps>

## Запуск службы

Мастер инициализации готовит новую службу на вашей системе. Каждая служба имеет свой собственный файл конфигурации, порт управления и уникальное имя. Запустите мастер инициализации с помощью этой команды:

```sh
alta init
```

:youtube[Мастер инициализации Cesbo Alta]{id="cnp3IsYHSZw"}

- **Имя службы**: Уникальный идентификатор для службы.
- **Путь к конфигурации**: Полный путь к файлу конфигурации; каталоги создаются по мере необходимости.
- **Логин администратора**: Имя пользователя для первого администратора.
- **Пароль администратора**: Установите и подтвердите пароль (не виден при вводе).
- **Порт управления**: Порт для API/веб-интерфейса (например, `127.0.0.1:8100`).
- **OTT Порт**: Порт для OTT стриминга (например, `127.0.0.1:8200`)
- **Добавить в systemd**: Регистрирует службу для управления системой.
- **Запуск при старте**: Включите, если добавлено в systemd.
- **Запустить службу**: Запуск сразу.

Вы можете запускать несколько служб, используя уникальные имена служб и порты. Подробнее: [Управление службами Alta](/en/alta/admin-guide/manage-services)

## Доступ к веб-интерфейсу

Ваша служба готова к использованию. Подключитесь к веб-интерфейсу, используя любой веб-браузер. Если вы оставили порт по умолчанию `8100`, адрес интерфейса будет `http://server-address:8100`, где server-address — это ваш IP-адрес или имя хоста сервера.
Для входа используйте имя пользователя `admin` и пароль, который вы задали в мастере.
