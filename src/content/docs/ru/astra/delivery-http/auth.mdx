HTTP авторизация контролирует доступ к вашему защищенному видеоконтенту. Она обеспечивает возможность просмотра ваших потоков только авторизованными пользователями.

![HTTP Auth](https://cdn.cesbo.com/help/astra/delivery/http-hls/auth/diagram.svg)

## Рабочий процесс авторизации

Прежде чем настроить авторизацию в Astra, необходимо разобраться, как работает этот процесс.

<Steps>

1. **Получение списка каналов от Middleware**

    ![Список телеканалов](https://cdn.cesbo.com/help/astra/delivery/http-hls/auth/step-1.svg)

    Клиент отправляет свои учетные данные в вашу систему Middleware. Если учетные данные верны, Middleware возвращает список каналов, к которым пользователь может получить доступ. Каждый канал включает уникальный токен для последующей авторизации.

2. **Запрос канала у Astra**

    ![Запрос канала](https://cdn.cesbo.com/help/astra/delivery/http-hls/auth/step-2.svg)

    Когда пользователь выбирает канал, его видеоплеер отправляет запрос в Astra с уникальным токеном из Шага 1.

3. **Astra авторизует запрос**

    Astra проверяет токен по настроенному методу авторизации. Если токен действителен, Astra возвращает поток канала. Если нет, система отвечает с ошибкой.

</Steps>

## Методы авторизации

- **[Авторизация пользователя](/en/astra/delivery-http/auth-user/)** - Использует логин/пароль или токены, хранящиеся в настройках пользователей Astra
- **[Авторизация через HTTP Backend](/en/astra/delivery-http/auth-backend/)** - Отправляет токен в вашу систему Middleware для проверки
- **[Securetoken](/en/astra/delivery-http/auth-securetoken/)** - Валидирует токены с использованием криптографических алгоритмов
- **[Авторизация по IP](/en/astra/delivery-http/auth-ip/)** - Проверяет, находится ли IP-адрес пользователя в вашем списке доступа

Выберите метод, который наилучшим образом соответствует вашим потребностям в безопасности и существующей инфраструктуре.
